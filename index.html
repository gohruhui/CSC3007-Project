About

Easy way to find information regarding Singapore's public housing transactions

Why

What is the long term price change / fluctuations on a particular HDB block?

What

Data

month, town, flat type, block, street name, storey range, floor area sqm, flat model, lease commence date, remaining lease and resale price

Data on hover

year-month, resale amount, storey range, total square meters, price per square feet, flat type, flat model of each transaction

How

The idiom chosen is a scatter plot which shows the resale transaction for the selected block. Each coloured point of varying size on the scatter plot indisactes a past resale transaction.

Critics

What is done poorly

Color and Size of each point does not mean anything.

Difficult to visually compare between transaction details.

Not responsive and does not fit the screen.

Users are not able to understand the trend.

User Interface / Idioms

01

02

Linear regression line - to show the trend of a property based on the variables

03

Each point can be coloured based on a colour spectrum (of light to dark colours), indicating the flat type

Interactions

01

Allows users to select up to 2 points to compare the resale transaction

02

Checkbox function can be implemented to allow user to filter based on story range

03

The coloured points (flat type) and filtering based on storey range allows a fair comparison of equal basis - for e.g. allows the user to further examine the flat model etc
